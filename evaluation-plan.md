### 2.4 调用方式

#### mobile-use

**方式一：CLI 命令（官方推荐）**

```bash
# 激活环境后直接调用
cd /tmp/mobile-use && source .venv/bin/activate
export ANDROID_HOME=/opt/homebrew/share/android-commandlinetools
export PATH=$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator:$PATH

# 基础任务
mobile-use "打开设置，查看当前电池电量，告诉我电量百分比"

# 带结构化输出
mobile-use "打开大众点评搜索深圳按摩，返回前3家店铺信息" \
  --output-description "A JSON list of objects, each with 'name', 'rating' and 'price' keys"

# 带测试记录（保存 trace）
mobile-use "打开小红书搜索美食" --test-name "xiaohongshu_search" --traces-path ./traces
```

CLI 完整参数：
- `--output-description, -o` — 指定结构化输出格式描述
- `--test-name, -n` — 测试名称（启用 trace 录制）
- `--traces-path, -p` — trace 保存路径（默认 `./traces`）
- `--with-video-recording` — 启用视频录制分析
- `--device-type, -d` — 设备类型（`local` / `limrun`）

**方式二：Docker 快速启动**

```bash
cd /tmp/mobile-use
bash ./mobile-use.sh \
  "打开设置查看电池电量" \
```

**方式三：Python SDK**

```python
import asyncio
from minitap.mobile_use.sdk import Agent
from minitap.mobile_use.sdk.types import ManualTaskConfig, PlatformTaskRequest

async def main():
    agent = Agent()
    await agent.init()
    result = await agent.run_task(
        request=PlatformTaskRequest(
            task=ManualTaskConfig(
                goal="打开设置查看电池电量",
                output_description="电池百分比",
            ),
        )
    )
    print(result)
    await agent.clean()

asyncio.run(main())
```

## 三、评测维度与指标

### 3.1 核心指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| **任务成功率 (SR)** | 任务是否正确完成 | 成功数 / 总数 × 100% |
| **任务耗时 (T)** | 从开始到完成的时间 | 秒 |
| **LLM 调用次数 (C)** | 总 LLM API 调用次数 | 次 |
| **操作准确率 (OA)** | 无误操作占比 | (总操作 - 误操作) / 总操作 × 100% |
| **Token 消耗 (TK)** | 总 input + output tokens | 千 tokens |

### 3.2 评判标准

**任务成功的定义**：
- 任务结果与预期完全一致（如正确的地址、正确的搜索结果）
- 不接受"部分完成"或"编造数据"
- 需通过人工验证（查看手机屏幕 + 核对返回结果）

**误操作的定义**：
- 点击了错误的元素
- 进入了错误的页面后需要返回
- 切回已离开的 App
- 输入了错误的文本
- 无效等待（超过 3 秒的不必要等待）

## 五、评测流程

### 5.1 每轮测试流程

```
1. 回到手机主屏幕（干净状态）
2. 关闭所有后台 App
3. 记录开始时间
4. 执行测试任务
5. 记录结束时间
6. 人工验证结果（查看手机屏幕 + 核对返回值）
7. 记录以下数据：
   - 任务是否成功（Success/Fail）
   - 总耗时（秒）
   - LLM 调用次数
   - 设备操作次数
   - 误操作次数
   - 返回结果的准确性
```

### 5.2 重复测试

每个测试用例运行 **3 次**，取中位数作为最终结果。原因：
- LLM 输出有随机性
- App 加载时间有波动
- 搜索结果可能变化

### 5.3 公平性保障

| 保障项 | 措施 |
|--------|------|
| 网络环境 | 同一 WiFi，同一时段测试 |
| 设备状态 | 每轮测试前重启 App、清理后台 |
| App 状态 | 不利用之前的搜索历史/缓存 |
| 评判标准 | 同一人评判所有项目 |
| 时间统计 | 统一使用脚本计时，包含启动到返回结果的全流程 |

## 十、注意事项

1. **模拟器检测**：小红书等 App 会检测模拟器，必须用真机测试
2. **登录状态**：小红书等需要登录才能使用，需提前登录好
3. **广告/弹窗**：首次打开 App 可能有广告弹窗，评测前先手动关闭
4. **搜索结果变化**：搜索结果可能因时间/地点变化，验证时以实际屏幕为准
5. **API 限额**：OpenAI 低 Tier 账户有 TPM 限制，需关注 rate limit
6. **Open-AutoGLM 部署**：需要 24GB+ GPU，可使用第三方 API（z.ai/Novita）替代自部署
